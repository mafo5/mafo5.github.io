What I've learned from Durchstarten mit HTML5

03.01.2015

generally

Regardless of the publishing date 2011 I have learned a lot from reading and refreshing the already known infos about HTML5. It has opened my mind for more semantical HTML and the usage of OOCSS. The JavaScript things are not so interesting cause Angular or even JQuery gives me a more extensive toolkit.

semantical HTML

Use section for recurring components and article for elements in text lists that can be accessed externally. You can use header and footer in each section or article. Each of this blocks resets the DOM structure so the browser can interpretate h1 as a subsection and not as a parallel section.

    <section>
        <header>
            <h1>Headline</h1>
        </header>
        <div>
            <p>Content</p>
        <div>
        <footer>
            <p>Copyright</p>
        </footer>
    </section>

time Element

In articles use time to define the time at which the text was created readable by machines.

  <time datetime="2015-01-03T19:20:00-01:00">
        Samstag, 03. Januar 2015, 19:20:00 MET
    </time>

limits of storage

The accessable size via technology is limited.

cookies
4 KB
userData
64 KB (trusted 640 KB)
Flash
100 KB (extendable via user confirmation every 10th multiplication)
SQLite
unlimit via Plugin
HTML5 Storage
5 MB (not extendable)

Cookies will be send with every HTTP Request!!!

Offline

Just with a defined manifest file you can tell the browser to save this and all listed HTML sites for offline usage.

This manifest file should be excluded by the server cache.

The site with the manifest defined will be always cached even if is not listed in the manifest!

HTML

  <html manifest="/cache.manifest">

Manifest

CACHE MANIFEST
# will save visited site for offline usage
FALLBACK:
/ /offline.html
NETWORK:
*

types of input

There are more types of input I have known.

But they are poorly supported by the browsers.

type="url"
for websites
type="range"
for sliders
type="date"
(or datetime, month, week and time) for date picker
type="search"
for clearable text inputs (with css limits)
type="color"
for html color picker

So with the exclution of color and url don't use them!!!

Microdata

Microdata is another form of microformats. It's declarting without usage of CSS classes.

This helps crawler like search engines to identify the content of the site and may give a proper summary.

Google has a massive support of those formats but doesn't actually ensure the usage of the defined informations.

<section itemscope itemtype="http://schema.org/Person">
    <img src="https://avatars1.githubusercontent.com/u/1301243?s=460" itemprop="image">
    <h1>Informationen</h1>
    <dl>
        <dt>Name</dt>
        <dd itemprop="name">Martin Fochler</dd>
        <dt>Work</dt>
        <dd itemprop="jobTitle">Developer</dd>
        <dt>Home</dt>
        <dd itemprop="address" itemscope itemtype="http://schema.org/PostalAddress">
            <span itemprop="addressLocality">Berlin</span>, <span itemprop="addressCountry">Germany</span>
        </dd>
    </dl>
    <h1>Digital Footprint</h1>
    <ul>
        <li><a href="https://mafo5.github.io/" itemprop="url">Github</a></li>
        <li><a href="https://twitter.com/illdvidd" itemprop="url">Twitter</a></li>
    </ul>
</section>
